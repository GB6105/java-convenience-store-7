요구사항

1. 입력한 상품의 가격, 수량을 기반으로 최종 결제 금액 계산
    상품별 가격과 수량을 곱함, 프로모션, 멤버십 할인 정책을 반영
2. 구매내역, 산출한 금액 정보를 영수증으로 출력
3. 추가 구매여부를 선택

재고 관리
1. 재고 확인
   -> 결제 가능 여부 반환
2. 재고 감소
    -> 결제 품목에 대한 재고 감소
3. 재고 정보 최신화

멤버십
1. 금액 할인
   프로모션 미 적용 금액에 대해 30% 할인 적용(최대 8000원)
    -> 항목중에 프로모션이 없는 경우메만 30%

영수증 출력
1. 영수증 출력
   - 구매 상품 내역
   - 증정 상품 내역
   - 금액 정보
     - 총 구매액
     - 행사 할인
     - 멤버십 할인
     - 내실 돈


서비스 흐름

입력 단계

- 상품 목록, 행사 목록 불러오기 (최초 1회)

1. 구매 상품 수량 입력
2. 프로모션 적용 가능 상품이 있을 경우
   -> 증정 받을 수 있는 상품 추가함
   -> 증정 받을 수 있는 상품 추가 하지 않음
3. 프로모션 재고가 부족할 경우
   -> 일부 수량에 대해 정가로 결제
   -> 프로모션 해당 개수만 결제
4. 멤버십 할인 적용 여부
   -> 멤버십 할인 적용함
    -> 멤버십 할인을 적용하지 않음
5. 전체 계산
6. 추가 구매 여부
   -> 업데이트된 상품 목록 호출
   -> 종료

구현 기능

### 파일 입출력
- 물품 목록 읽어오기
  -> 재고 정보로 저장하기
- 물품 목록 덮어쓰기
  - 개수 변경
- 프로모션 읽어오기

### 재고 관리
- 재고 정보 저장하기
- 재고 정보 조회하기
- 재고 정보 수정하기
- 최종 정보로 물품 목록 수정하기

### 프로모션 관리
- 프로모션 정보 

### 입력
- 구매 상품, 수량 입력
- 프로모션 적용 상품 수량 추가 여부 입력
- 프로모션 재고 부족 시 일부 수량 정가 결제 여부 입력
- 멤버십 할인 적용 여부 입력
- 추가 구매 여부 입력

### 출력
- 안내 메시지 출력
- 물품 목록 출력
- 프로모션 적용 상품 안내메시지 출력
- 프로모션 재고 부족 시 정가 결제 여부 안내메시지 출력
- 멤버십 할인 적용 여부 확인 문구 출력
- 영수증 출력
  - 편의점 이름 
  - 상품명, 수량, 금액
  - 증정(프로모션)항목 상품명, 수량
  - 총 구매액, 수량, 금액
  - 행사할인  금액(음수)
  - 멤버십할인 금액(음수)
  - 내실돈 금액(양수)
- 추가 구매 여부 확인 안내 문구 출력
